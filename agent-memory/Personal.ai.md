# Personal.ai

Identity-based AI platform where users build a "Personal Language Model" from their own data. The most-funded pure consumer play in user-integrated AI memory.

## Basics

- **Funding**: $23.8M total -- $7.8M seed (Differential VC, Supernode Global), $14M subsequent round, plus crowdfunding on Republic. Backed by a16z, Gaingels, Mana Ventures.
- **Founded**: 2020, San Diego
- **Founder**: Suman Kanuganti (CEO)
- **Users**: 80,000+

## What It Does

Users feed information -- notes, conversations, documents, messages -- into their "Personal AI." The system builds a persistent, evolving model that reflects the user's knowledge, voice, and preferences. The AI can then respond on the user's behalf in messaging contexts.

Key features:
- **Personal Language Model**: adapts to user's knowledge and communication style
- **Memory Stacks**: organized collections of user-provided information
- **AI Copilot**: responds in user's voice in messaging platforms
- **Memory scoring**: shows how much the AI has learned about different topics

## Why It Matters

Personal.ai represents the best-funded attempt at user-owned AI memory. But it's a **walled garden**: the memory lives inside Personal.ai's own model and doesn't travel to Claude, Cursor, or other agents. Users build memory *for* Personal.ai, not *across* their AI tools.

This is the opposite of the portable, agent-agnostic memory that the [[User-Integrated Memory Gap]] describes. Personal.ai solves the "my AI should know me" problem but not the "my memory should work everywhere" problem.

The same walled-garden pattern appears at platform scale in [[Platform Memory Features|ChatGPT Memory, Claude memory, and Gemini memory]].
