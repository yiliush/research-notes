# Khoj

Open-source, self-hostable AI "second brain." The closest venture-funded product to filling the [[User-Integrated Memory Gap]].

## Basics

* **GitHub**: khoj-ai/khoj -- 34,741 stars, 2,210 forks (verified May 28 2026)

* **License**: AGPL-3.0

* **Created**: August 2021

* **Funding**: $500K from Y Combinator (S23 batch)

* **Founders**: Debanjum Singh Solanky, Saba Imran

## What It Does

Personal AI that searches across private documents -- PDFs, Markdown, Notion, Word, images -- and combines semantic search with conversational AI. Self-hostable or cloud-hosted.

Key properties:

* **Works with any LLM**: local models via Ollama or cloud APIs (GPT, Claude, Gemini)

* **Self-hostable**: user owns all data, runs on own infrastructure

* **Multiple interfaces**: web, Obsidian plugin, Emacs package, WhatsApp, desktop

* **Agentic features**: web search, scheduled automations, custom agents, deep research

## Why It Matters

Khoj occupies a unique position: the only YC-backed product that gives users a self-hostable personal knowledge base working across multiple AI providers. It bridges the gap between [[Obsidian Vaults as Agent Memory|grassroots vault-based memory]] and [[Memory Middleware Players|developer middleware]].

Unlike [[Mem0]] or [[Zep]] (developer APIs), the user is the customer. Unlike Obsidian vault patterns, it has real retrieval infrastructure -- semantic search, not brute-force file loading.

**Limitation**: positioned as a "second brain" / personal AI assistant, not as a memory layer for other agents. Khoj knowledge doesn't automatically flow into Claude Code or Cursor sessions. The user interacts *with* Khoj, not *through* Khoj into other agents.
