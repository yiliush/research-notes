# Pieces

On-device AI productivity platform for developers. The most-funded product that connects user-controlled memory to multiple AI agents.

## Basics

- **Funding**: $21.5M total -- $13.5M Series A led by Drive Capital (July 2024), with Cintrifuse Capital and RedHawk Ventures
- **Founded**: 2022, Cincinnati
- **Founders**: Tsavo Knott (CEO), Mark Widman (CTO)
- **Product**: desktop app + IDE extensions + CLI (primarily closed-source)

## What It Does

Captures developer workflow context across applications -- IDE activity, browser research, terminal commands, meeting notes -- and builds persistent memory spanning up to 9 months of activity. Runs entirely on-device.

Key properties:
- **On-device processing**: no data leaves the machine
- **Long-term memory**: captures and indexes 9 months of workflow activity
- **Multi-agent integration**: connects personal context to Copilot, Claude, Cursor
- **Time-based retrieval**: "what was I working on last Tuesday?" queries

## Why It Matters

Pieces is the most-funded product that bridges personal context and AI agents. It solves a real version of the [[User-Integrated Memory Gap]]: your coding agent doesn't know what you were doing yesterday, but Pieces does, and it can inject that context.

However, Pieces is a *developer productivity tool* that happens to have memory, not a *memory system* that happens to serve developers. The memory is captured passively from workflow, not actively curated by the user.

GitHub presence is modest (647 stars for docs, 231 for open-source components) -- the product is primarily closed-source.
