# Trag

Trag (usetrag.com / tragai.cc) is a minimalist AI code reviewer built around one idea: code style and policy rules written in plain English, not regex or AST patterns. Think of it as a "linter with judgment" — you describe a rule in a sentence, and Trag enforces it on every PR.

## Origin

Small/early-stage company, limited public funding disclosure. Founder details are not heavily publicized — Trag has stayed product-focused rather than building a founder brand like [[Greptile]].

## Funding & Traction

- No notable disclosed funding round.
- Free Hobby tier (unlimited repos, unlimited engineers, 14-day Team trial).
- Customer logos not prominent.

## Product

- Natural-language rule creation. Example: "warn if any function exceeds 50 lines" or "flag use of deprecated `lodash.get`."
- Automated PR review using custom rules.
- AI-driven fix suggestions (no direct commits).
- GitHub and GitLab support.
- Multi-language semantic analysis.

## Pricing

- Hobby: free, with unlimited repos and engineers, 14-day Team trial.
- Paid tiers exist but aren't prominently published — Trag pushes the free entry hard.

## Differentiators

- **Custom rules in plain English.** This is the core wedge — turns code review into a configurable linter with semantic understanding.
- **Hobbyist-friendly free tier.** Unlimited engineers + repos free is unusually generous.
- **Low pricing friction.** Easier to try than enterprise-style alternatives.

## Risk factors

- Tiny relative to category leaders. Brand awareness minimal.
- Natural-language rules are a feature [[Ellipsis]] and [[CodeRabbit]] can copy easily.
- No notable funding signal means runway is uncertain.
